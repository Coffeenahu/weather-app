## 1. 개요

### 1.1 목적

할 일 목록을 브라우저에서 관리할 수 있는 간단한 Todo 앱을 제작한다.
서버나 프레임워크 없이 HTML/CSS/JS만으로 동작하며, 새로고침 후에도 데이터가 유지된다.

### 1.2 배경

- 초보자 학습용 프로젝트로, 기본적인 DOM 조작·이벤트 처리·LocalStorage 사용법을 익힌다.
- 별도의 빌드 도구 없이 파일을 열기만 해도 즉시 실행 가능해야 한다.

### 1.3 관련 문서

- 기술 스택: `CLAUDE.md`
- 구현 코드: `index.html`, `css/style.css`, `js/main.js`

## 6. 아키텍처 결정

### 6.1 프로젝트 레벨

| 레벨 | 특징 | 선택 |
|------|------|:----:|
| **Starter** | 단순 구조, 빌드 도구 불필요 | ✅ |
| Dynamic | BaaS 연동 | ☐ |
| Enterprise | 마이크로서비스 | ☐ |

### 6.2 핵심 기술 결정

| 결정 | 선택 | 이유 |
|------|------|------|
| 언어 | Vanilla JavaScript | 초보자 친화적, 의존성 없음 |
| 스타일 | 순수 CSS | 빌드 없이 즉시 실행 |
| 데이터 영속성 | LocalStorage | 서버 없이 간단하게 저장 |
| 상태 관리 | 배열 변수 + render() 함수 | 단순하고 명확한 단방향 흐름 |

### 6.3 폴더 구조

```
my-todo/
├── index.html        ← UI 구조
├── css/style.css     ← 스타일
├── js/main.js        ← 상태 + 렌더링 + 이벤트
├── images/           ← (미사용, 예비)
└── docs/
    ├── 01-plan/
    │   └── features/my-todo.plan.md   ← (현재 문서)
    └── 02-design/
```
